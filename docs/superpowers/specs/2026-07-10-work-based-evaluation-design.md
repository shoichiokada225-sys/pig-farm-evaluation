# 作業評価（大項目→小項目）モード 設計

作成日: 2026-07-10 / 対象: `~/pig-farm-evaluation/index.html`

## 目的

既存の「豚舎別 10項目 × レベル1〜5」評価に加え、**作業（大項目）を選び、その作業の評価観点（小項目）をレベル1〜5で採点する**新しい評価軸を追加する。現場の実作業に即した具体的な評価を可能にする。

出典:
- 睦沢農場「業務の目的と注意点」pptx（44作業／7カテゴリ、目的・注意点・よくあるミス付き）
- swine-manual（技術ガイド）は基準文の裏取り素材として参照

## ユーザー体験

入力タブ上部に評価モード切替を置く: **[豚舎別評価] [作業評価]**

作業評価モードの流れ:
1. **カテゴリを選ぶ** — 飼養管理 / 衛生管理 / 繁殖管理 / 分娩管理 / 施設管理 / 記録管理 / 出荷管理
2. **作業＝大項目を選ぶ** — そのカテゴリの作業（例: 除フン、給餌、AI注入…）
3. **観点＝小項目を採点** — 各観点を「基準を見る」で開き、レベル1〜5の行動基準文をタップして採点

評価者名・被評価者名・評価日・全体所感・進捗バー・基準アコーディオンは既存UIを流用する。

## データ構造 — `work-criteria.json`（新設・正本）

```jsonc
{
  "commonAspects": [            // 全作業共通の観点テンプレート（5つ）
    { "id": "care",     "name": "丁寧さ・確実さ",      "levels": ["…L1", "…L2", "…L3", "…L4", "…L5"] },
    { "id": "time",     "name": "時間内の遂行",        "levels": [...] },
    { "id": "tools",    "name": "器具・設備の扱い",     "levels": [...] },
    { "id": "report",   "name": "報告・記録",          "levels": [...] },
    { "id": "observe",  "name": "作業外の観察（豚の異常察知）", "levels": [...] }
  ],
  "categories": [
    { "id": "feeding", "name": "飼養管理" }, ...
  ],
  "works": [
    {
      "id": "removal-dung",
      "no": "No.12",
      "category": "衛生管理",
      "name": "除フン",
      "purpose": "…（pptx由来・基準を見る際の補助表示）",
      "aspects": [               // 作業固有の観点（1〜3個）。共通観点は自動で後ろに付く
        { "id": "order",  "name": "除糞→洗浄→消毒の順序", "kanten": "…", "levels": [L1..L5] },
        { "id": "underfloor", "name": "床下・ピットの確認", "kanten": "…", "levels": [L1..L5] }
      ]
    }
  ]
}
```

- **共通観点（commonAspects）**: レベル基準文は作業名プレースホルダ `{work}` を含むテンプレート。表示時に作業名へ置換。5観点×5レベル＝25文を1セット定義し、全44作業に適用（品質を揃え、データ量を圧縮）。
- **作業固有観点（works[].aspects）**: pptx の目的・注意点・よくあるミスを素材に、作業ごとに1〜3個の観点＋レベル1〜5基準文を生成。
- 各作業の採点対象 = 作業固有観点（先頭）＋ 共通観点5個。1作業あたり6〜8観点。

### レベルの意味（既存と統一）
1=要指導 / 2=要改善 / 3=標準 / 4=上回る / 5=模範的

## 生成方針

44作業の固有観点＋基準文は、カテゴリ単位でサブエージェントに分担生成させ（7カテゴリ→7体並列）、全件を人手（Claude）で検証・統合する。共通観点テンプレートは本設計者が直接執筆（品質の要）。

## ストレージ / 機能統合

- localStorage キー: `pig_work_eval_data_v1`（評価データ）。作業評価は豚舎に依存しない全社共通。
- 評価レコード: `{ id, date, evaluator, evaluatee, workId, workName, category, scores:{aspectId:1..5}, comments:{aspectId}, overall, createdAt }`
- **履歴**: 作業評価の履歴は被評価者＋作業名＋日付で一覧。既存の豚舎別履歴とはモードで切り替え表示。
- **グラフ**: 観点別レーダー（選んだ作業の観点で）。
- **CSV**: 作業評価用に別出力（作業名・観点列）。
- **全体バックアップ**: 既存 `exportAll`/`importAll` に `workEval` セクションを追加し、1ファイルで全部入り。
- **下書き自動保存/復元**: 作業評価モードも対応（キーに workId を含める）。

## UI 実装方針

- 既存の `buildCards()` は豚舎別評価用のまま。作業評価は新関数 `buildWorkCards()` で構築。
- 基準アコーディオン・スコアボタン・進捗バーのCSS/挙動（`setScoreUI`/`updProg`/`toggleCrit`）は共通化して流用。
- モード状態 `evalMode`（'barn' | 'work'）を localStorage 保持。モードで豚舎バー/カテゴリ・作業セレクタを出し分け。

## 多言語

UI文言（モード名・カテゴリ・観点名）は TX に4言語キーを追加。作業固有の基準文は当面 ja のみ（データ構造は `levels` を将来 `{ja,en,vi,id}` に拡張可能な形にしておくが、初版は文字列配列）。

## テスト

Playwright スモーク（`smoke.js`）に作業評価フローを追加:
- モード切替 → カテゴリ→作業選択で観点が出る
- 基準タップ採点 → 進捗 → 保存 → 履歴1件 → レーダー
- バックアップに workEval が含まれ復元できる
- 既存の豚舎別評価が壊れていない（回帰）
