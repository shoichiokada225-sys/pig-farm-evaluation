# 豚舎別 従業員評価システム（HSS実技試験）

養豚場で働く従業員を上司（管理者）が評価するためのスマホ対応 Web アプリ（PWA）。
評価モードは2つ:

1. **豚舎別評価** — 「🏠 豚舎」セレクターで **分娩舎 / ストール舎 / 肥育舎 / 子豚舎（離乳舎）** を切替、各舎10項目をレベル1〜5で採点
2. **作業評価** — カテゴリ（飼養/衛生/繁殖/分娩/施設/記録/出荷）→ 作業（大項目・44作業）を選び、その作業の観点（小項目=固有観点＋共通観点5つ）をレベル1〜5で採点。出典は睦沢農場「業務の目的と注意点」

- 各舎の評価項目は A「舎別業務」7項目 ＋ B「仕事への姿勢」3項目（全舎共通）
- **評価基準の内蔵**: 各項目の「▼ 基準を見る」で評価の観点＋レベル1〜5の行動基準を表示。基準文をタップするとそのまま採点される（`criteria-data.json` と同一の正本）
- 採点進捗バー（ヘッダー下に固定表示、n/10）
- 下書きの自動保存＋リロード・豚舎切替後の自動復元
- 舎ごとにデータを別々に保存（評価が混ざらない）。分娩舎は従来データを継承
- 切替で未保存の入力がある場合は確認ダイアログを表示
- 全豚舎の評価データ＋項目の一括バックアップ／復元（設定タブ。復元はマージ方式＝既存の記録は残る）
- レーダーチャートは直近と前回の2回分を重ね描き
- PWA: ホーム画面に追加可・オフライン動作（Chart.js もローカル同梱）

## 対応言語

- 日本語 (JP)
- English (EN)
- Tiếng Việt (VI)
- Bahasa Indonesia (ID)

ヘッダー右上の言語ボタンで切替。選択した言語は localStorage に保存される。

## 動作確認手順

1. `index.html` をブラウザで開く
2. Chrome DevTools（F12）→ デバイスツールバー（Ctrl+Shift+M）でスマホ表示を確認

### 入力画面

1. 評価日・評価者名・被評価者名を入力
2. 10項目それぞれで 1〜5 のスコアボタンをタップ → 色が変わる
3. 任意でコメントを入力
4. 「保存」→ トースト「保存しました」
5. スコア未入力で保存 → 警告＋未入力項目にスクロール
6. 「リセット」→ 確認後クリア

### 編集機能

1. 履歴画面で評価をタップ → 詳細モーダル
2. 「編集」ボタン → 入力画面にデータが読み込まれる
3. 画面上部にオレンジの「編集中」バナー表示
4. 修正後「更新」ボタンで上書き保存
5. 「取消」ボタンで編集を中断

### 履歴画面

- 被評価者フィルター、タップで詳細、削除、CSV出力

### グラフ画面

- 被評価者選択 → 平均点折れ線グラフ ＋ レーダーチャート

### 評価項目の編集・他PCへの共有（編集ツール）

評価項目（セクション・項目名・説明）は「設定」タブ（パスワード保護）で編集できる。
編集した項目を**別のPCでも同じ内容で使う**には、ファイルの書き出し／読み込みを使う。

1. 編集した側のPC：設定タブ → 「⬇ 項目を書き出し（ファイル保存）」→ `hss_eval_items_YYYYMMDD.json` が保存される
2. そのファイルを相手のPCへ渡す（USB・メール・LINE・共有フォルダ等、手段は自由）
3. 受け取った側のPC：設定タブ → 「⬆ 項目を読み込み（ファイルから）」→ ファイルを選択
4. 確認ダイアログで OK → 評価項目が上書きされ、入力画面に即反映される

- サーバー不要・ネット接続不要。どのPC・ブラウザでも動作する
- 読み込みは形式チェック済み（不正なファイルは「ファイルの形式が正しくありません」で拒否）
- 共有されるのは**評価項目の定義のみ**。各PCの評価データ（履歴）は端末ごとのまま

## 評価基準（レベル1〜5）の管理

### 豚舎別評価の基準
基準の**正本は `criteria-data.json`**。ここから3つの成果物を生成する:

| 成果物 | コマンド | 用途 |
|---|---|---|
| アプリ内の基準表示 | `python build_html.py` | index.html に埋め込み |
| 評価基準表 PDF（A3横） | `python build_pdf.py` | 印刷・配布用 |
| 評価基準表 Excel | `python build_xlsx.py` | 編集・回覧用 |

基準を修正したら `criteria-data.json` を編集 → 上記3スクリプトを再実行する。
アプリの基準はデフォルト項目（item1〜7=舎別業務、item8〜10=仕事への姿勢）に対応。設定タブで追加したカスタム項目には基準ボタンは表示されない。

### 作業評価の基準
正本は2ファイル。編集後は `python build_html.py` で再埋め込みする:

- `work-criteria.json` — 共通観点5つ（丁寧さ/時間/器具/報告/観察。`{work}` が作業名に置換される）とカテゴリ定義
- `works-data.json` — 44作業（大項目）と作業固有観点（各2〜3個、レベル1〜5基準文つき）

## ファイル構成（レイヤ）

```
index.html      マークアップ＋埋め込みデータ（criteriaData / workCriteriaData）
styles.css      スタイル一式
js/util.js      汎用ユーティリティ（toast/esc/sanitizeId）
js/i18n.js      UI文言辞書（ja/en/vi/id）と t()
js/data.js      静的データ層：豚舎定義・評価基準・作業カタログのアクセサ
js/store.js     永続化層：localStorageキー・状態・評価データ・バックアップ
js/ui.js        描画層：カード/基準/履歴/詳細/グラフ/CSV/設定画面
js/app.js       アプリ層：初期化・モード/豚舎/作業切替・入力/保存/編集フロー
```

読み込みは util → i18n → data → store → ui → app の順（後のレイヤほど前に依存する）。

## 技術仕様

- 静的ファイルのみ（サーバー不要・file://直開きでも動作）＋ PWA（`manifest.json` / `sw.js` / アイコンは `build_icons.py` で生成）
- Chart.js v4 はローカル同梱（`chart.umd.min.js`）— オフラインでも動作
- Service Worker はネットワーク優先＋キャッシュフォールバック（https配信時のみ登録）
- 豚舎切替: `localStorage`（`pig_eval_active_barn`）。舎ごとに保存キーを分離（`pig_eval_<舎>_data_v1` / `pig_eval_<舎>_items_v1`、分娩舎のみ従来キー）
- データ保存: `localStorage`（分娩舎の評価データ: `pig_farm_evaluation_v1` / 評価項目: `pig_farm_eval_items_v1`）
- 言語設定: `localStorage`（キー: `pig_farm_eval_lang`）
- 評価項目の共有: 書き出し/読み込み（JSON、`_type:"pig_farm_eval_config"` 形式・サーバー不要）
- 全データバックアップ: `_type:"pig_farm_eval_backup"` 形式。復元は評価をIDでマージ（既存優先）、項目は上書き
- 自動保存: 300ms デバウンス。下書きはリロード・豚舎切替後に自動復元
- テスト: `smoke.js`（Playwright、50項目）

## テスト実行

```bash
node smoke.js   # playwright が必要（別プロジェクトの node_modules を参照している場合はパスを調整）
```
